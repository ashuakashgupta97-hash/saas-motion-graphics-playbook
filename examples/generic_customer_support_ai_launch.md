# Example Adaptation: Customer Support AI Launch

This example shows how the same playbook can be adapted to a completely different story.

## Use Case

An organization already has a support ticketing platform. The missing layer is not submission, but smarter triage, suggested responses, and priority prediction.

## Narrative Arc

### Problem

Support workflows are digital, but agents still spend time:

- reading long tickets
- identifying urgency
- spotting repeat issues
- routing tickets to the right queue

### AI Layer

AI adds:

- suggested priority
- category recommendation
- summary and context extraction
- escalation risk flag
- explanation for the recommendation

### Proof Scenes

- incoming ticket text -> summarized issue card
- routing lanes -> recommended team destination
- ticket detail modal -> explanation panel
- KPI dashboard -> faster triage and lower backlog

## Why It Maps Well

The same scene system still applies:

- title reveal
- workflow baseline
- decision complexity
- AI overlay
- reasoning modal
- engine/process diagram
- impact KPI close

That is why this repo is useful beyond a single workflow.
