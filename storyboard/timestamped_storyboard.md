# Timestamped Storyboard

## Example 120-Second Structure

| Scene | Time | Purpose | Voiceover beat |
| --- | --- | --- | --- |
| 1 | `0:00-0:06` | Hero title reveal | product name + tagline |
| 2 | `0:06-0:16` | Scale of the workflow | volume, teams, operating context |
| 3 | `0:16-0:28` | Decision complexity | list of checks/questions |
| 4 | `0:28-0:40` | Existing workflow baseline | digital system exists, but judgment still manual |
| 5 | `0:40-0:55` | AI layer reveal | recommendations, amounts, reasoning, overlays |
| 6 | `0:55-1:08` | Reasoning / trust moment | modal, explanation, transparency |
| 7 | `1:08-1:24` | Engine or process logic | agent flow, backend intelligence, audit trail |
| 8 | `1:24-1:40` | Proof scene 1 | document understanding / classification / extraction |
| 9 | `1:40-1:52` | Proof scene 2 | route/path/policy/exception logic |
| 10 | `1:52-2:00` | Impact and close | KPI outcomes + final positioning |

## Notes

- the “before” state should not always mean paper chaos
- if a digital workflow already existed, say so clearly
- impact metrics should appear when spoken, not earlier
